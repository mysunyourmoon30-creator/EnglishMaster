Prepare a clean commit summary for the Content Review and Approval Workflow.

Check:
- dotnet build passes
- dotnet test passes
- no temporary files
- no secrets
- no unnecessary generated files
- documentation updated
- admin routes updated
- permissions updated
- existing modules still work

Then provide:
1. Git status summary
2. Suggested commit message
3. Pull request title
4. Pull request description
5. Testing evidence
6. Security evidence
7. Risks or limitations

Do not create new features.
