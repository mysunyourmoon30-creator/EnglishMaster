# v0.3.0 Test Strategy

## Testing Strategy Summary

v0.3.0 needs focused tests around delivery behavior, certificate ownership, public verification safety, analytics privacy, and authorization.

## Unit Tests

- Email delivery result mapping.
- Retry eligibility rules.
- Certificate completion rule evaluation.
- Certificate verification code generation constraints.
- Analytics aggregation calculations.
- Template data validation.

## Integration Tests

- Email delivery create/send/fail/retry flow using a fake sender.
- Notification-to-email delivery flow.
- Certificate generation for completed course.
- Student certificate list scoping.
- Admin certificate management.
- Analytics endpoints and summaries.

## Authorization Tests

- Email settings require permission.
- Email retry requires permission.
- Notification delivery admin actions require permission.
- Certificate admin actions require permission.
- Student certificate list requires authentication.
- Admin analytics require reporting/analytics permission.

## Public Verification Safety Tests

- Valid certificate verification returns only public-safe fields.
- Invalid code returns safe not-found response.
- Revoked certificate returns safe status.
- Verification does not expose internal IDs, private learner data, admin notes, or raw completion records.

## Email Sender Mock Tests

- Successful provider send.
- Provider failure.
- Timeout/cancellation behavior.
- Sanitized provider error storage.
- Retry after failure.
- Secret masking in settings/status responses.

## Certificate Generation Tests

- Generate certificate once for completed course.
- Prevent duplicate certificates.
- Deny generation when completion rules are not met.
- Verify ownership and admin access boundaries.
- Handle generation failure safely.

## Analytics Query Tests

- Admin metrics are bounded and permission-protected.
- Student trend summary is scoped to current student.
- Analytics omit sensitive metadata and auth fields.
- Export, if included, is capped and permission-protected.

## Release Test Gates

- `dotnet build --no-restore`.
- `dotnet test --no-restore`.
- Focused integration tests for email, certificates, verification, and analytics.
- Manual staging smoke for provider configuration, email test send, certificate generation, verification, and analytics views.
