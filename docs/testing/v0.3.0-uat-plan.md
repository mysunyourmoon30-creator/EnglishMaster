# v0.3.0 UAT Plan

## Objective

Confirm v0.3.0 is usable, secure, and release-ready for notification delivery, certificate issuance and verification, and advanced analytics workflows.

## Release Candidate

`v0.3.0-rc.1`

## Test Roles

- SuperAdmin.
- Admin with notification and certificate permissions.
- Admin with `reports.read`.
- Admin without `reports.read`.
- Student with completed course progress.
- Public visitor.

## Test Data Required

- One active email provider configuration in staging using test credentials.
- At least one queued notification and one failed notification delivery record.
- One active certificate template.
- One published and active course with completed progress for a student.
- One issued valid certificate and one revoked or invalid verification scenario.
- Student learning activity, quiz attempts, practice sessions, progress, streak, and certificate data for analytics.

## Notification Delivery Scenarios

- Review queued notification deliveries.
- Confirm delivery attempts are visible to authorized admin users.
- Trigger resend or retry for a failed delivery.
- Confirm unauthorized users cannot manage delivery records.
- Confirm external provider secrets are not visible in UI, API responses, logs, or docs.

## Certificate Scenarios

- Create, edit, activate, and deactivate a certificate template.
- Confirm `certificates.read` users can view templates but cannot write.
- Confirm `certificates.manage` users can create and update templates.
- Generate a certificate for a completed course as the owning learner.
- Confirm a learner cannot generate a certificate for another learner.
- Confirm duplicate generation returns or preserves the existing certificate instead of issuing uncontrolled duplicates.
- Verify a valid certificate through the public verification page.
- Verify invalid, unknown, and revoked certificate states.
- Confirm public verification does not expose internal ids or rendered certificate body content.

## Analytics Scenarios

- Open admin analytics as a user with `reports.read`.
- Confirm admin analytics shows aggregate metrics only.
- Confirm a user without `reports.read` is denied admin analytics.
- Open student analytics as an authenticated learner.
- Confirm student analytics is scoped to the current learner.
- Test default date range and a custom bounded date range.
- Confirm public users cannot access analytics APIs.

## Regression Smoke Scenarios

- Login and logout.
- Admin navigation loads.
- Student navigation loads.
- Public home/search still loads.
- Existing reporting route still enforces `reports.read`.
- Health endpoints work in staging.

## Acceptance Criteria

- No Blocker defects remain open.
- No High security, privacy, public exposure, or delivery-secret exposure defects remain open.
- Notification delivery behavior is accepted by the release owner.
- Certificate generation and public verification behavior are accepted by the release owner.
- Analytics privacy and permission checks pass.
- Known limitations are accepted by the release owner.
- Rollback and backup plans are ready.

## Defect Severity Levels

- Blocker: Data loss, security breach, complete outage, or release cannot proceed.
- High: Critical workflow blocked, private data exposed, or delivery secrets exposed.
- Medium: Important workflow impaired with a workaround.
- Low: Cosmetic, copy, or minor usability issue.
