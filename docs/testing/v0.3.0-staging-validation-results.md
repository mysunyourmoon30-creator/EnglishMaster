# v0.3.0 Staging Validation Results

## Status

`Partially Run - Blocking Defects Found And Fixed`

## Summary

Staging validation for `v0.3.0-rc.1` was executed on 2026-07-17 against a **local, non-Docker Staging-configured deployment**, not the documented `docker-compose.staging.yml` path. Docker was unavailable on the validation machine (not installed, and a prior attempt to enable virtualization in BIOS/UEFI failed — the firmware settings screen could not be reached), so validation used the fallback documented in `docs/deployment/local-run.md` ("If Docker is unavailable, continue using the normal .NET local workflow"): `dotnet run` for both API and Web with `ASPNETCORE_ENVIRONMENT=Staging`, against a fresh LocalDB database (`Server=(localdb)\mssqllocaldb;Database=EnglishMasterStaging`).

This was the first time in the project's history that the application was run end-to-end against a freshly migrated database. That surfaced **9 real defects** (see `docs/testing/v0.3.0-defect-log.md`, V030-DEF-001 through V030-DEF-009), 8 of which were fixed and verified in this session. All 8 fixes are application/migration-code defects that would reproduce identically under the Docker Compose path — none are artifacts of skipping Docker. One defect (V030-DEF-009, no certificate admin UI) remains open pending a product decision.

## Decision

Current staging result: `Partially Run - Blocking Defects Found And Fixed`.

Eight Blocker-severity defects were found and fixed in this session (V030-DEF-001 through 008); the fixes are in the working tree but **not yet committed or reviewed**. One High-severity gap remains open (V030-DEF-009, no certificate admin UI) and needs a product/scope decision before Certificate Scenarios can be exercised.

Final tag and production go-live remain blocked until:
1. The fixes in this session are reviewed, committed, and (ideally) covered by regression tests — especially the migration corrections, since a missing/broken migration is exactly the kind of defect unit/integration tests should catch but evidently did not.
2. Remaining validation matrix items (notification delivery, student analytics, security/privacy role-matrix spot checks) are executed with real test fixtures.
3. V030-DEF-009 is triaged (build the UI, or explicitly accept API-only certificate testing for this release).
4. Full UAT (`docs/testing/v0.3.0-uat-plan.md` scenarios, `docs/testing/v0.3.0-uat-execution-record.md`) is completed and signed off.
