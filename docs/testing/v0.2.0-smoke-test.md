# v0.2.0 Smoke Test

## Preconditions

- API and Web are deployed to the target environment.
- Database migration and seed process completed.
- SuperAdmin account is available.
- Media and publishing storage paths are configured.

## Manual Smoke Tests

| Area | Steps | Expected Result |
| --- | --- | --- |
| Login | Open `/login`, sign in as SuperAdmin. | Redirects to `/admin`; user menu appears. |
| Admin dashboard | Open `/admin`. | Dashboard loads without errors. |
| Public home | Open `/`. | Home page renders. |
| Public search | Open `/search?q=a`. | Search page loads and returns safe empty or result state. |
| Dictionary | Open `/dictionary`. | Dictionary discovery page loads. |
| Course detail | Open `/learn/courses`. | Course discovery route loads. |
| Lesson detail | Open `/learn/lessons`. | Lesson discovery route loads. |
| Quiz practice | Open `/learn/practice`. | Practice dashboard loads or safe error state appears if unauthenticated. |
| Student dashboard | Open `/learn/dashboard`. | Recommendations, goals, study plan, motivation, and weekly card load or safe empty states appear. |
| Practice session | Generate/start practice from `/learn/practice`. | Session opens when due items exist. |
| Study plan | Open `/learn/study-plan`, generate today's plan. | Plan appears with items or safe empty state. |
| Recommendations | Open `/learn/recommendations`. | Recommendation groups load. |
| Achievements | Open `/learn/achievements`. | Achievement list loads. |
| Weekly report | Open `/learn/reports/current-week`, generate report. | Weekly report appears with metrics and insights. |
| Admin content workflow | Create/edit a word, lesson, course, book, or quiz. | Admin API accepts valid changes and rejects invalid input. |
| Publishing preview | Open `/admin/publishing`. | Publishing entry page and job pages load. |
| Import validation | Upload a small CSV import and validate it. | Rows and validation errors are shown safely. |
| Content quality check | Open `/admin/content-quality`, run or view checks. | Quality page loads and protected API responds. |
| Bulk operation | Open `/admin/bulk-operations`. | List/create pages load. |
| Reports | Open `/admin/reports`. | Report metrics load for authorized user. |
| Notifications | Open `/admin/notifications` and `/learn/notifications`. | Notification pages load. |
| Health checks | Call `/health/live` and `/health/ready`. | Live returns healthy; ready returns healthy when database is reachable. |

## Pass Criteria

No P0 or P1 issue is found. P2 issues must have an owner and release decision.
