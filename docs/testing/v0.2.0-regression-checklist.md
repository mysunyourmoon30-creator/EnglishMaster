# v0.2.0 Regression Checklist

## v0.1.0 Features

- Solution restore, build, and tests.
- Admin login/logout.
- Core admin dashboard.
- Words, categories, tags, media, pronunciation, grammar, lessons, courses, books, and quizzes.
- Security roles, users, and permissions.

## Admin Modules

- Publishing.
- Reports.
- Notifications and email messages.
- Content quality.
- Content revisions and restore requests.
- Bulk operations.
- Import jobs and validation.
- Achievement definitions API.

## Public Modules

- Home page.
- Search page.
- Dictionary, grammar, books, quizzes, course, and lesson route shells.
- Public search hides non-public content.

## Student Modules

- Dashboard.
- Recommendations.
- Practice.
- Learning goals.
- Study plan.
- Motivation.
- Achievements.
- Activity.
- Weekly learning reports.

## Security Regression Checks

- Admin APIs require authentication and permissions.
- `/api/v1/me/*` APIs require authentication.
- Learner-owned data is not visible across users.
- Quiz correct answers are not exposed through public or learner DTOs.
- Upload/import path safety checks reject unsafe names.

## Performance Regression Checks

- List endpoints remain paginated or bounded.
- Search and recommendation endpoints apply limits.
- Report history is paginated.
- Admin recent activity is bounded.
- No obvious full entity graph is loaded for list views.
