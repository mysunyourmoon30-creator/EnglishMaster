# v0.2.0 UAT Plan

## Objective

Confirm v0.2.0 is usable, secure, and release-ready for admin content operations, public discovery, and authenticated learner workflows.

## Test Roles

- SuperAdmin.
- Reviewer.
- ContentEditor.
- Student.
- Public visitor.

## Test Data Required

- At least one word, category, tag, pronunciation, grammar topic, grammar rule, lesson, course, book, quiz, and media item.
- One published and one unpublished item for public safety checks.
- One student account with practice, goals, study plan, activity, achievements, and weekly reports.
- One import CSV with valid rows and one with invalid rows.

## Admin Scenarios

- Manage words, categories, tags, media, pronunciations, grammar, lessons, courses, books, and quizzes.
- Review reports, notifications, email messages, content quality, content revisions, bulk operations, imports, and publishing.
- Verify permission-restricted APIs reject users without the needed permission.

## Reviewer Scenarios

- Review content quality and content revision records.
- Confirm restore requests can be approved or rejected by authorized users only.

## ContentEditor Scenarios

- Create and edit content within assigned permissions.
- Confirm forbidden operations return safe access denied responses.

## Student Scenarios

- View dashboard and recommendations.
- Generate practice items and complete a practice session.
- Create goals and generate a study plan.
- Record or view motivation activity and achievements.
- Generate and review weekly learning reports.

## Public Visitor Scenarios

- Open public home and search.
- Confirm unpublished, draft, archived, or private content does not appear.
- Confirm quiz answer keys are not visible.

## Acceptance Criteria

- P0 and P1 defects are closed.
- Required smoke tests pass.
- Security and privacy checks pass.
- Known limitations are accepted by release owner.
- Rollback and backup plans are ready.

## Defect Severity Levels

- P0: Data loss, security breach, complete outage.
- P1: Critical workflow blocked or private data exposed.
- P2: Important workflow impaired with workaround.
- P3: Cosmetic, copy, or minor usability issue.

## Sign-Off Checklist

- [ ] Product owner sign-off.

- [ ] Security/privacy sign-off.
- [ ] Operations sign-off.
