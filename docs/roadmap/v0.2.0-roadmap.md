# v0.2.0 Roadmap

## Theme

v0.2.0 introduces student-facing public learning pages for EnglishMaster. The release should make published learning content browsable and readable without requiring login, while preserving strict safety boundaries around unpublished content, private metadata, and quiz answers.

## Included Scope

- Public course pages.
- Public lesson pages.
- Public dictionary pages.
- Public grammar pages.
- Public book pages.
- Public quiz pages.
- Public route documentation.
- Public API documentation.
- Public DTO safety rules.
- Published content visibility rules.
- Media rendering rules.
- Quiz correct-answer safety rule.
- Authenticated practice and spaced repetition foundation.
- Authenticated learning goals and daily study plan foundation.
- Streaks, achievements, and motivation foundation.
- Student weekly learning report foundation.

## Safety Requirements

- Public queries return only published public content.
- Public DTOs are separate from admin DTOs and persistence models.
- Nested related content is filtered independently.
- Unpublished, archived, private, missing, or future scheduled content returns a not-found result on detail pages.
- Media rendering never exposes private paths or raw storage keys.
- Public quiz payloads never include correct answers, correctness flags, or answer-key metadata.
- Practice quiz review payloads do not expose quiz correct answers.
- Study plan quiz items do not expose quiz correct answers.
- Motivation metadata strips obvious sensitive auth fields.

## Intentionally Not Included

The following are out of scope for v0.2.0:

- Student progress.
- Login-based learning.
- Certificates.
- AI tutor.
- Personalized recommendations.
- AI-generated practice content.
- Enrollment flows.
- Payment or gated content access.
- Full quiz attempt history.

## Basic Reporting And Admin Dashboard Analytics

Purpose: Basic Reporting and Admin Dashboard Analytics gives authorized administrators read-only operational visibility into platform health.

Basic Reporting and Admin Dashboard Analytics is the next recommended v0.2.0 feature after the public learning documentation baseline.

The feature should provide authorized administrators with a read-only dashboard for operational visibility across content, learning progress, quiz activity, and recent activity.

### Reporting Scope

- Reporting module application service or equivalent read model.
- Read-only reporting API endpoints.
- Blazor admin reports page at `/admin/reports`.
- Admin dashboard metrics.
- Content status metrics.
- Learning progress metrics.
- Quiz analytics metrics.
- Recent activity.
- Permission enforcement using `reports.read`.

### Reporting API

Recommended endpoint set:

```text
GET /api/v1/reports/admin-dashboard
GET /api/v1/reports/content-status
GET /api/v1/reports/learning-progress
GET /api/v1/reports/quiz-analytics
GET /api/v1/reports/recent-activity
```

All reporting endpoints should require `reports.read`.

### Reporting UI

The Blazor admin reports page should include summary metric cards, content status, learning progress, quiz analytics, recent activity, and loading, empty, error, and forbidden states.

### Reporting Security And Privacy

Reporting should require authentication and `reports.read`. It should return aggregate data by default, avoid unnecessary personal data, avoid quiz answers and sensitive assessment details, and avoid exposing internal errors or infrastructure details.

### Reporting Performance Limitations

The first reporting implementation should use bounded read-only queries. Avoid unbounded recent activity lists and avoid loading full entity graphs for aggregate metrics.

If performance becomes a concern, follow-up work should evaluate indexes, cached summaries, background aggregation, or a dedicated reporting store.

### Reporting Intentionally Not Included Yet

- Advanced analytics.
- Charts library integration.
- Data warehouse integration.
- AI insights.
- Export reporting.

## Known Limitations

- Public learning pages are read-only unless a safe quiz submission flow is added.
- No student-specific state is persisted.
- No learner dashboard is included.
- Public routes assume published slugs are stable.
- Quiz feedback requires a server-side submission endpoint before interactive checking is safe.
- Practice tables still need a reviewed EF migration.
- Practice generation is rule based and does not include AI prompts.
- Learning goal and study plan tables still need a reviewed EF migration.
- Study plan generation is rule based and does not include AI or calendar integration.
- Motivation is rule based and does not include AI, leaderboards, or social features.
- Weekly learning reports are generated on demand and do not include AI or external analytics.

### Motivation Limitations

- Automatic activity recording is not wired into every workflow.
- No leaderboard or social features.
- No AI motivation.

## Student Weekly Learning Report

Student Weekly Learning Report adds authenticated learner summaries across study time, daily plans, lessons, courses, books, practice, quizzes, goals, streaks, and achievements.

### Scope

- WeeklyLearningReport and WeeklyLearningReportInsight records.
- Authenticated `/api/v1/me/learning-reports` endpoints.
- Learner pages at `/learn/reports`, `/learn/reports/current-week`, and `/learn/reports/{id}`.
- Student dashboard Weekly Report card.
- Rule-based insights for inactivity, low study time, practice due, low quiz score, streak, achievements, and goals.

### Limitations

- No EF migration generated yet.
- No background scheduled report generation.
- No AI-generated summaries.
- No external analytics service.
- No export or email delivery.

## Next Recommended Development Step

Generate and review EF migrations for learner-facing progress modules, then add automatic activity recording from real lesson, course, book, quiz, practice, goal, study plan, and report workflows.

## Notification And Email Foundation

The notification and email foundation adds in-app notification records, queued email messages, role-based permissions, API endpoints, admin pages, and a learner notification page.

Recommended endpoint set:

```text
GET  /api/v1/me/notifications
GET  /api/v1/me/notifications/unread-count
POST /api/v1/me/notifications/{id}/read
GET  /api/v1/admin/notifications
GET  /api/v1/admin/email-messages
POST /api/v1/admin/email-messages
```

The first implementation uses a development email sender and exposes service abstractions for later workflow integration.

## Next Recommended Development Step

Integrate notification creation into content review, publishing, user management, and quiz workflows. Keep real email provider integration deferred until retry, secret storage, delivery monitoring, and provider selection are defined.

## Content Quality And QA

Content Quality adds rule-based QA checks for existing content modules. It includes quality rules, persisted quality checks, findings, scores, admin pages, and permissions.

The system is intentionally deterministic and does not include AI QA in v0.2.0.

## Next Recommended v0.2.0 Feature

Add content versioning and revision history so editors can compare changes, recover prior versions, and audit content updates before publishing.

## Content Versioning And Revision History

Content Versioning adds a simple revision timeline for content changes, sanitized snapshots, restore requests, admin pages, permissions, and API endpoints.

The feature is intentionally not an advanced audit platform or visual diff tool. It records safe JSON snapshots and supports a reviewer-approved restore request workflow.

### Content Versioning Scope

- ContentRevision records.
- ContentRevisionRestoreRequest records.
- Revision and restore API endpoints.
- Blazor admin revision pages.
- Revision History links from Word, Lesson, Course, Book, and Quiz detail pages.
- Snapshot safety rules.
- Permissions for read, request, approve, and manage actions.

### Content Versioning Limitations

- Automatic capture is not wired into every content command yet.
- Restore completion records workflow state but does not yet mutate the original content record.
- Advanced visual diff is deferred.

## Next Recommended Development Step

Wire automatic revision capture into Word, Lesson, Course, Book, Quiz, and PublishTemplate create/update/publish workflows, then add command-specific restore execution for the same modules.

## Content Bulk Operations

Content Bulk Operations add a synchronous admin workflow for applying simple actions to multiple content items and recording per-item results.

The feature includes BulkOperation and BulkOperationItem records, API endpoints, permissions, Blazor admin pages, and tests for successful, failed, and partially completed runs.

### Bulk Operation Limitations

- Content list checkbox integration is deferred.
- Export does not yet generate downloadable files.
- Background queue execution is intentionally excluded until retry, monitoring, and cancellation semantics are defined.

## Next Recommended v0.2.0 Feature

Add automatic revision capture for core content commands, then add content list checkbox selection for bulk operation creation.

## Advanced Import Validation

Advanced Import Validation adds import job history, row-level validation, preview, confirmation, Word import execution, Word rollback, import permissions, API endpoints, and Blazor admin pages.

### Advanced Import Scope

- Import jobs, rows, and validation errors.
- CSV and JSON upload through the admin import UI.
- Validation and preview for Words, Categories, Tags, Grammar Topics, Grammar Rules, Lessons, Courses, Books, and Quizzes.
- Confirm and run lifecycle controls.
- Word import execution and rollback.
- Validation error export.
- Permission enforcement with `import.read`, `import.upload`, `import.validate`, `import.run`, and `import.rollback`.

### Advanced Import Limitations

- Dedicated run and rollback behavior is implemented for Words first.
- Other content types validate and preview, but their run behavior remains deferred.
- CSV parsing is simple and should be replaced before large or complex migration files.
- Background processing is intentionally deferred until retry, monitoring, idempotency, and cancellation behavior are designed.

## Next Recommended v0.2.0 Feature

Add dedicated import runners and rollback support for Categories, Tags, Lessons, Courses, Books, and Quizzes, then replace the simple CSV parser with a robust CSV library-backed parser.

## Better Public Search And Discovery

Better Public Search adds a unified learner-facing `/search` route backed by anonymous public API endpoints.

### Public Search Scope

- Unified `/search` page.
- Public search API under `/api/v1/public/search`.
- Filters for keyword, content type, CEFR, category, and tag where supported.
- Public-safe result DTOs.
- Active/published visibility enforcement.
- Simple suggestions endpoint.
- Integration tests for public safety, filtering, pagination, aliases, special characters, and quiz answer safety.

### Public Search Limitations

- Relevance is simple and database-backed.
- No external search engine is included.
- No typo tolerance, stemming, synonym support, or advanced ranking.
- Tags currently apply to Word results.
- Cross-type search uses bounded per-type queries for MVP.

## Next Recommended v0.2.0 Feature

Add dedicated public detail pages for search result types that are not fully learner-facing yet, then add server-side quiz submission with answer-key safety tests.

## Learning Recommendations And Continue Learning

Learning Recommendations add authenticated learner endpoints and pages for continuing in-progress content and showing rule-based recommendations.

### Scope

- Continue learning for lessons, courses, and books.
- Recommended courses, lessons, words, grammar, and quizzes.
- Review recommendations from low quiz scores.
- Next lesson in course.
- `/learn/dashboard` and `/learn/recommendations`.
- `/api/v1/me/learning/*` endpoints.

### Limitations

- Recommendations are rule-based and simple.
- No AI, ML, or external recommendation service is included.
- SQL migration for the new progress foundation should be generated before database deployment.
- Category/tag inference from learning history is limited.

## Next Recommended v0.2.0 Feature

Generate the learning progress database migration, then add richer progress capture from real lesson, course, book, and quiz interactions.
