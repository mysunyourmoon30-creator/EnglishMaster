## Queue Rules

- Do not run later prompts if an earlier prompt is running or failed.
- Run design prompts before implementation prompts.
- Run review prompts before docs and stabilization prompts.
- Run build/test after implementation and review prompts.
- Keep commit/tag/deploy actions manual and explicitly approved.
- Do not start v0.3.0 implementation until the v0.2.0 release gate is accepted by the release owner.
