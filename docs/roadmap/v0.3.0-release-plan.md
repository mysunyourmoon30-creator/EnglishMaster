# v0.3.0 Release Plan

## Release Goal

Ship a controlled v0.3.0 release focused on real notification delivery, completion evidence, and privacy-safe insights.

## Release Phases

| Phase | Outcome | Release Gate |
| --- | --- | --- |
| Email Provider Foundation | Provider-backed email can send, fail safely, and retry | Tests pass and secrets are masked. |
| Notification Delivery Integration | Notifications can use email delivery with status tracking | Authorization and retry tests pass. |
| Certificate Foundation | Students/admins can use issued completion evidence | Ownership, duplicate, and permission tests pass. |
| Certificate Verification | Public verification is safe and useful | Public DTO safety tests pass. |
| Analytics Foundation | Admin/student insights are clearer and bounded | Performance, privacy, and authorization tests pass. |
| Stabilization | v0.3.0 is ready for RC review | Full build/test/security/docs pass. |

## Release Risks

- Email provider secrets or provider errors leaking.
- Duplicate or forged certificates.
- Public verification exposing private learner information.
- Analytics causing expensive queries or exposing sensitive details.
- Scope creep into AI Tutor, payment, marketplace, mobile, or external search.

## Required Evidence Before RC

- `dotnet restore`.
- `dotnet build --no-restore`.
- `dotnet test --no-restore`.
- Focused module tests for email, notifications, certificates, verification, and analytics.
- Security audit.
- Staging validation plan.
- Updated known limitations.
- Rollback and hotfix notes.

## Release Candidate Recommendation

Prepare v0.3.0 RC only after all selected modules meet acceptance criteria and no Blocker or High security/privacy defects remain.
