# v0.3.0 MVP Slice

## MVP Objective

Ship a compact v0.3.0 release that makes EnglishMaster more useful in real operation: users receive real notifications, students can prove course completion, and admins/students get clearer learning signals.

## Slice 1: Email Delivery Module

Must have:

- Provider abstraction for external email delivery.
- Configurable SMTP or provider-backed implementation.
- Delivery status model.
- Failed delivery capture.
- Manual retry for failed delivery.

Should have:

- Email test send page.
- Provider health check.

Could have:

- Improved templates.
- Weekly learner summary email.

## Slice 2: Notification Delivery Module

Must have:

- Notification delivery settings.
- Link notifications to email delivery where appropriate.
- Delivery status surfaced in admin views.
- Manual retry path for failed notification email.

Should have:

- Delivery filters by status.
- Delivery status summary.

Could have:

- Additional notification templates.

## Slice 3: Certificate Module

Must have:

- Certificate template foundation.
- Course completion certificate generation.
- Student certificate page.
- Admin certificate management page.

Should have:

- Verification code.
- Public certificate verification page.

Could have:

- Certificate PDF polish.

## Slice 4: Analytics Improvement Module

Must have:

- Admin metrics improvements.
- Student learning trend summary.
- Basic event taxonomy review.
- Privacy-safe aggregation.

Should have:

- Export analytics CSV.

Could have:

- Admin analytics filters.

## Release Shape

The MVP should be shippable if only the must-have items are completed. Should-have and could-have items must not block release unless explicitly promoted by the release owner.
