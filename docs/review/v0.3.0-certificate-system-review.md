# v0.3.0 Certificate System Review

Status: Completed

## Review Scope

- Certificate template management endpoints and permissions.
- Course completion certificate generation flow.
- Issued certificate persistence and uniqueness constraints.
- Public certificate verification API and Web page.

## Findings

- No release-blocking issues found.
- Public verification exposes only certificate verification fields and does not require authentication.
- Learner certificate generation is scoped to the authenticated user and requires completed published course progress.
- Duplicate generation returns the existing certificate, preserving one certificate per learner/course.
- Revoked certificates verify as invalid rather than disappearing from public history.

## Follow-up Notes

- Future admin revocation endpoints should keep the existing public verification behavior: revoked certificates remain discoverable but invalid.
- If certificate templates become brand-specific, template selection should move from first active template to explicit course/program configuration.
