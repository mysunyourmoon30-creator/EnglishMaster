# v0.3.0 Security And Permission Review

Status: Completed

## Review Scope

- Certificate template admin APIs.
- Learner certificate generation and certificate list APIs.
- Public certificate verification API.
- Admin and learner analytics APIs.
- Permission registration and documented route expectations.

## Findings

- No release-blocking security or permission defects were found.
- Admin certificate template reads require `certificates.read`.
- Admin certificate template writes, activation, and deactivation require `certificates.manage`.
- Learner certificate APIs require authentication and derive the learner from the current user claim.
- Certificate generation validates completed course ownership before issuing a certificate.
- Public certificate verification is intentionally anonymous and returns only verification code, recipient name, course title, template code, issue date, revocation status, and validity.
- Admin analytics requires `reports.read` and returns aggregate metrics only.
- Learner analytics requires authentication, derives the learner from the current user claim, and does not accept a user id parameter.
- Analytics date ranges are bounded by the application layer to prevent unbounded query windows.

## Residual Notes

- Public certificate verification does not currently have a route-specific rate limit. This is acceptable for the v0.3.0 release candidate because the endpoint exposes only public verification data, but rate limiting should be considered before broad production exposure.
- Certificate rendered bodies remain available only through authenticated learner certificate responses. Public verification does not expose rendered certificate content.
- Analytics remains read-only in this scope. Future detailed drilldowns should repeat this review before adding personally identifiable data or learner-level admin exports.
