# v0.4.0 Architecture Guardrails

## Status

`Draft - Not Approved`

## Purpose

Set architecture guardrails for future v0.4.0 planning. This document does not authorize implementation.

## Guardrails

- Preserve Clean Architecture boundaries.
- Keep domain rules out of Blazor components and endpoint handlers.
- Keep infrastructure provider details behind application abstractions.
- Require explicit permission policies for admin APIs.
- Scope learner APIs to the authenticated user.
- Keep public APIs on learner-safe DTOs.
- Add integration tests for new user-facing API behavior.
- Add focused UI tests or manual QA evidence for high-risk workflows.

## Data And Security

- Do not expose secrets in UI, API responses, logs, docs, or test snapshots.
- Do not add cross-user query paths without ownership filters.
- Do not add public endpoints without response-shape review.
- Prefer bounded queries and pagination for collection endpoints.

## Planning Result

These guardrails are ready for v0.4.0 planning review.
