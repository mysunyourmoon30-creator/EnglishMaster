## Safe Final Tag Commands

Do not execute automatically:

```powershell
git tag -a v0.3.0 -m "EnglishMaster v0.3.0 final release"
git push origin v0.3.0
```

## Decision

Do not create or push the final `v0.3.0` tag until:
1. The release owner triages V030-DEF-009 (no certificate admin UI) — build it, or explicitly accept API-only certificate management for this release. See `docs/release/v030-def-009-certificate-admin-ui-scope.md` for a scoped breakdown (template-CRUD-only vs. a full admin-wide issued-certificates browser) and a recommendation.
2. Staging UAT is re-run against the actual `docker-compose.staging.yml` path at least once (this pass used a local non-Docker workaround; the container/networking layer is unvalidated).
3. Production readiness, backup/rollback readiness, and release-owner/operations sign-off are complete.
