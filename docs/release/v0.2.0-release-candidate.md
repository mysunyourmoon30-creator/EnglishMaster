# v0.2.0 Release Candidate

## Feature Summary

EnglishMaster v0.2.0 expands the platform from admin content management into learner-facing discovery, practice, planning, motivation, reporting, quality, revision, import, notification, publishing, and operational workflows.

## Major Modules Included

- Public search and learner discovery.
- Learning recommendations and continue learning.
- Practice system and spaced repetition foundation.
- Learning goals and daily study plan.
- Streaks, achievements, and motivation foundation.
- Student weekly learning reports.
- Admin reporting dashboard.
- Notification and email foundation.
- Content quality checks.
- Content versioning and restore requests.
- Bulk operations.
- Advanced import validation.
- Publishing jobs, templates, and artifacts.
- Security, roles, permissions, and production hardening.

## Security Notes

- Admin API endpoints require authentication and module permissions.
- `/api/v1/me/*` endpoints require authentication and scope data to the current user or student profile.
- Public search exposes only active or published learner-safe content.
- Quiz correct answers must not be exposed in public or learner report payloads before a server-side submission flow.
- Sensitive metadata is stripped from motivation activity logs.
- No secrets should be committed.

## Known Limitations

- Several learner progress modules still need reviewed EF migrations before deployment.
- Learner activity capture is not wired into every workflow.
- Weekly reports are generated on demand, not by a scheduler.
- Public detail pages are still discovery-oriented and rely on search entry points where dedicated detail pages are not yet implemented.
- Achievement definition admin editing UI is deferred; protected API coverage exists.
- AI, external analytics, mobile-specific flows, payments, marketplace behavior, and microservices are intentionally excluded.

## Required Manual Testing

- Login and logout.
- Admin dashboard and major admin navigation.
- Public search with filters and empty results.
- Learner dashboard, recommendations, practice, goals, study plan, motivation, achievements, activity, and weekly reports.
- Report generation and regeneration.
- Admin reporting, notifications, content quality, revisions, bulk operations, import jobs, and publishing jobs.
- Permission-limited admin access with a non-superadmin account.
- Quiz payload inspection to confirm answer-key safety.

## Deployment Notes

- Configure `ConnectionStrings:DefaultConnection` for staging and production.
- Review and apply EF migrations in a controlled environment.
- Configure media and publishing storage paths.
- Configure secure cookies and HTTPS in non-development environments.
- Use the existing health endpoints for readiness checks.

## Rollback Notes

- Keep the previous deployed build and database backup available.
- Roll back application binaries first when no migration has been applied.
- If migrations are applied, use reviewed database backup or a tested down-migration plan.
- Pause background or manual import/publish operations before rollback.

## Open Risks

- Migration review remains the largest release blocker.
- Integration tests run in Testing mode with automatic non-security API authentication, so manual auth checks remain required.
- NuGet vulnerability checks warn in restricted environments when the feed is unavailable.
- Some public route pages are stabilization entry points and should be replaced with richer dedicated detail pages later.
