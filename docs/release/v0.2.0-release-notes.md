## Release Summary

v0.2.0 moves EnglishMaster beyond the MVP admin foundation into a broader release candidate with learner-facing discovery, planning, practice, motivation, reporting, and operational workflows.

## Major Modules

- Public search and public route entry points.
- Student learning dashboard, recommendations, practice, goals, study plan, motivation, achievements, activity, and weekly learning reports.
- Admin reporting, publishing, content quality, content revisions, bulk operations, import validation, notifications, and email foundations.
- Security roles, permissions, authenticated admin APIs, and learner-owned `/me` APIs.

## Student Features

- Learning dashboard and continue-learning entry points.
- Learning recommendations.
- Practice sessions and spaced repetition foundation.
- Learning goals and daily study plan.
- Streaks, achievements, motivation activity, and weekly reports.
- Public discovery routes for search, dictionary, grammar, books, courses, lessons, and quizzes.

## Admin Features

- Admin dashboard and reporting modules.
- Word, grammar, lesson, course, book, quiz, media, category, tag, and pronunciation management.
- Achievement definition API foundation.
- Notifications and email message management foundation.

## Content Operations Features

- Publishing now includes jobs, templates, and published artifacts.
- Content quality checks.
- Content revision history and restore requests.
- Bulk operations.
- Advanced import validation and path-like filename rejection.

## Security Improvements

- Admin APIs require permission policies.
- Learner APIs scope data to the authenticated user or student profile.
- Public search uses learner-safe DTOs and published/active content filters.
- Quiz answers are not exposed through public search or weekly learning reports.
- Import upload rejects path-like file names.
- Media upload validates size, content type, file signature, and file names.

## Operations Improvements

- Health endpoints are available for liveness and readiness.
- Deployment and operations runbooks are documented.
- Release candidate, smoke, UAT, staging validation, go-live, rollback, hotfix, and post-release monitoring docs are available.

## Known Limitations

See [v0.2.0 Known Limitations](v0.2.0-known-limitations.md).

## Upgrade Notes

- Review and apply EF Core migrations in a controlled environment.
- Configure production/staging connection strings and storage paths through secrets or environment configuration.
- Verify seeded roles and permissions after deployment.
- Run staging validation, the v0.2.0 smoke test, permission spot checks, and UAT before go-live.

## Rollback Notes

Use the [v0.2.0 Rollback Plan](v0.2.0-rollback-plan.md). Database and file-storage backups must be taken before applying migrations or replacing production binaries.

## Release Status

The final release package is prepared, but production go-live remains blocked until staging/manual validation, backup readiness, and operations sign-off are complete.
