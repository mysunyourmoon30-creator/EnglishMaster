# v0.3.0 Release Closure

## Release Version

v0.3.0

## Release Status

`Tagged (2026-07-18); production go-live remains No-Go`

The v0.3.0 final tag has been created and pushed. Staging UAT was executed (against a local non-Docker path — see "Known Limitations" below), all defects found were fixed and verified, backup/rollback readiness was verified with a real drill, operations sign-off documentation was verified current, and self-contained monitoring/alerting plus an SMTP adapter were built and verified. Production **deployment** (distinct from the tag) is not approved because operational/organizational gates remain incomplete — see `docs/release/v0.3.0-production-go-live-checklist.md`.

## What Was Delivered

- External email provider boundary (MailKit-based SMTP adapter, Gmail-compatible) and automatic notification delivery queue (`EmailDeliveryWorker`).
- Notification retry, failure handling, and admin review documentation.
- Certificate template management, including a full admin CRUD UI (built post-tag-prep, closing V030-DEF-009's core gap).
- Learner certificate generation.
- Public certificate verification.
- Admin advanced analytics.
- Student analytics.
- Self-contained system health monitoring dashboard and threshold-based email alerting (`SystemHealthWorker`), built without dependency on any specific hosting platform.
- Security, permission, regression, RC, UAT, staging, go-live, smoke, and monitoring documentation.
- A new regression test (`FreshDatabaseMigrationTests`) closing the gap that let 8 Blocker defects through undetected in the first place.

## What Was Deferred

- Production deployment itself (tag ≠ deployment).
- Docker Compose staging re-validation — attempted, abandoned due to a real hardware/firmware boot-stability issue on the development machine, not pursued further.
- ~~Route-specific rate limiting for public certificate verification~~ — **resolved as a post-tag follow-up** (10 requests/minute per IP, verified with real 429 responses). See `docs/operations/certificate-system.md`.
- An admin-wide "list all issued certificates across students" browser (only self-scoped `/me/certificates` exists) — deferred, needs a scope decision.
- Analytics rollups or warehouse integration — not needed at current data volume.
- Production-scale email provider (SendGrid/SES/Mailgun with SPF/DKIM) — Gmail SMTP is set up for low volume; a dedicated provider is a separate decision if volume grows.
- Any v0.4.0 feature implementation — `docs/roadmap/v0.4.0-scope-lock.md` is a draft, not yet approved.

## Open Defects

No confirmed open product defects are recorded. All 10 defects found during the 2026-07-17/18 staging validation pass (8 Blocker + 2 Low/High) are fixed and verified — see `docs/testing/v0.3.0-defect-log.md`.

## Known Limitations

- Docker Compose container/networking parity remains unvalidated on this hardware (see above).
- File storage backup was verified with a single small test file — see `docs/testing/v0.3.0-backup-restore-verification.md` — but not against a non-trivial production-scale media library.
- No named operations owner or on-call rotation exists yet — organizational, not technical.

## Final Recommendation

The `v0.3.0` tag is created; that gate is closed. Keep production **deployment** as `No-Go` until the operational items in `docs/release/v0.3.0-production-go-live-checklist.md` (named ops owner, production backup scheduling, production-scale email provisioning if needed, formal operations/rollback sign-off) are complete.
