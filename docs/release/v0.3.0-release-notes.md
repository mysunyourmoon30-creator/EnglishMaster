## Release Summary

v0.3.0 adds notification delivery infrastructure, learner certificates, public certificate verification, and advanced analytics foundations for admins and students.

## Notification Delivery

- External email provider boundary for provider-backed delivery.
- Notification delivery queue for pending outbound work.
- Retry and failure handling for delivery attempts.
- Admin delivery review and resend workflow.
- Operations documentation for delivery checks.

## Certificate System

- Certificate template management APIs and admin UI.
- Certificate generation for learners who complete eligible published courses.
- Learner certificate list.
- Public certificate verification API and page.
- Certificate operations and API documentation.

## Advanced Analytics

- Admin analytics overview with aggregate learner and platform metrics.
- Student analytics overview scoped to the authenticated learner.
- Date range handling with bounded query windows.
- Admin and student analytics UI documentation.

## Security Improvements

- New certificate permissions: `certificates.read` and `certificates.manage`.
- Certificate learner endpoints derive ownership from the authenticated user.
- Public certificate verification exposes only public verification fields.
- Admin analytics uses existing `reports.read`.
- Student analytics does not accept a caller-provided user id.

## Verification

- `dotnet restore`
- `dotnet build`
- `dotnet test`
- v0.3.0 notification, certificate, analytics, security, and regression review documents.

## Known Limitations

- Staging UAT has not been completed.
- No v0.3.0 production deployment has been performed.
- Email provider settings must be configured outside the repository.
- Public certificate verification does not yet have route-specific rate limiting.
- Analytics use live EF aggregates; rollups remain future work if measured latency requires them.

## Release Status

The v0.3.0 release candidate package is prepared for UAT. Final release tagging and production deployment remain blocked until manual staging validation and release owner approval are complete.
