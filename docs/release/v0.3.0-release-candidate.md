# v0.3.0 Release Candidate

## Feature Summary

EnglishMaster v0.3.0 expands the platform with production-oriented notification delivery, certificate issuance and public verification, and advanced learning analytics.

## Major Modules Included

- External email provider abstraction and implementation boundary.
- Notification delivery queue.
- Notification retry, failure tracking, and resend support.
- Certificate template management.
- Learner certificate generation from completed courses.
- Public certificate verification page and API.
- Admin advanced analytics overview.
- Student analytics overview.
- v0.3.0 security, permission, and regression verification evidence.

## Security Notes

- Admin notification and email operations remain protected by notification/email permissions.
- Certificate template reads require `certificates.read`.
- Certificate template writes require `certificates.manage`.
- Learner certificate APIs require authentication and derive ownership from the current user claim.
- Public certificate verification is anonymous by design and returns only public verification fields.
- Admin analytics requires `reports.read` and returns aggregate metrics.
- Student analytics requires authentication and does not accept caller-provided user ids.

## Required Manual Testing

- Admin notification delivery queue review and resend flow.
- Email provider configuration in staging using non-production test credentials.
- Certificate template create, edit, activate, and deactivate workflow.
- Learner certificate generation after course completion.
- Public certificate verification with valid, invalid, and revoked certificates.
- Admin analytics dashboard with a user that has `reports.read`.
- Admin analytics forbidden state with a user that lacks `reports.read`.
- Student analytics dashboard for an authenticated learner.
- Navigation visibility for notification, certificate, and analytics entry points.

## Deployment Notes

- Configure email provider settings through staging or production secrets.
- Keep email sending disabled or pointed at a test provider until staging UAT approves delivery behavior.
- Review EF migrations before applying them to shared environments.
- Verify seeded permissions after deployment.
- Use existing health checks for readiness validation.

## Rollback Notes

- Keep the previous deployed build and database backup available.
- If migrations are applied, rollback requires a reviewed database backup or tested down-migration path.
- Pause notification delivery jobs before rollback if delivery workers are enabled.
- Disable external provider delivery before restoring older application binaries.

## Open Risks

- Public certificate verification does not yet have route-specific rate limiting.
- Email provider behavior still requires staging validation with real provider credentials.
- Analytics are live EF aggregates and may need rollups after production-scale measurement.
- Manual UAT, deployed health checks, backup readiness, and operations sign-off remain required before production release.
