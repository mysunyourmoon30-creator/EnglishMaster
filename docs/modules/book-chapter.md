# Book Chapter

## Purpose

`BookChapter` is the ordered child entity inside a `Book`. It gives books a structured table-of-contents shape without creating a separate top-level business module. Chapters are managed from the Book detail page and through nested Book API routes.

## BookChapter Fields

| Field | Description |
| --- | --- |
| `Id` | Unique chapter identifier. Generated by the domain and configured with `ValueGeneratedNever` in EF Core. |
| `BookId` | Required parent book identifier. |
| `Title` | Required chapter title. Trimmed and used to generate `Slug`. |
| `Slug` | URL-friendly value generated from `Title`. |
| `Summary` | Optional short summary. Stored as an empty string when omitted. |
| `ContentMarkdown` | Optional markdown content for the chapter. Stored as an empty string when omitted. |
| `SortOrder` | Display order inside the book. Must be zero or greater. |
| `Lessons` | Ordered `BookChapterLesson` entries that connect the chapter to lessons. |
| `IsActive` | Indicates whether the chapter is active. Deletes deactivate the chapter. |
| `CreatedAt` | Chapter creation timestamp. |
| `UpdatedAt` | Chapter last update timestamp. |

## Book Relationship

`BookChapter` belongs to one `Book`.

- `Book` owns the `Chapters` collection.
- EF Core maps `Book` to many `BookChapter` records with cascade delete from book to chapter rows.
- The Application layer validates that the parent book exists and is active before a chapter can be added.
- Chapter ordering is controlled by `SortOrder`.

## Lesson Relationship

`BookChapter` connects to lessons through `BookChapterLesson`.

- A chapter can contain many lessons.
- One lesson can appear in many book chapters.
- Response DTOs include chapter lessons enriched with lesson title, slug, summary, and CEFR level.
- Lesson add/remove/reorder behavior is managed through chapter-specific API routes.

See [Book Chapter Lesson](book-chapter-lesson.md).

## Domain Rules

- `Id` and `BookId` cannot be empty GUIDs.
- `Title` is required, trimmed, and length-limited to 200 characters.
- `Slug` is generated from `Title` and length-limited to 220 characters.
- `Summary` is optional and length-limited to 1000 characters.
- `ContentMarkdown` is optional and length-limited to 12000 characters.
- `SortOrder` must be zero or greater.
- `Activate` and `Deactivate` update `IsActive` and `UpdatedAt`.
- `Reorder` changes `SortOrder` and updates `UpdatedAt`.
- Duplicate lessons inside a chapter are prevented by `AddLesson` returning the existing relation.
- Domain code does not reference EF Core, ASP.NET Core, Blazor, or infrastructure.

## API Endpoints

| Method | Route | Purpose |
| --- | --- | --- |
| `GET` | `/api/v1/books/{bookId}/chapters` | List chapters in a book. |
| `POST` | `/api/v1/books/{bookId}/chapters` | Add a chapter to a book. |
| `POST` | `/api/v1/books/{bookId}/chapters/reorder` | Reorder all chapters in a book. |
| `GET` | `/api/v1/book-chapters/{id}` | Get one chapter by id. |
| `PUT` | `/api/v1/book-chapters/{id}` | Update a chapter. |
| `DELETE` | `/api/v1/book-chapters/{id}` | Deactivate a chapter. |

Full status codes and examples are documented in [Book Chapter API](../api/book-chapter-api.md).

## Request Examples

Create a chapter:

```json
{
  "title": "Getting Started",
  "summary": "Set up the topic",
  "contentMarkdown": "## Getting Started",
  "sortOrder": 0
}
```

Reorder chapters:

```json
{
  "orderedChapterIds": [
    "bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb",
```

## Blazor UI

Book chapters are managed inline on `/admin/books/{id:guid}`.

- Admins can add a chapter.
- Admins can edit a chapter.
- Admins can delete a chapter, which deactivates it.
- Admins can reorder chapters using Up and Down actions.
- Loading, empty, error, and validation states are shown through the Book detail page.

## Test Coverage

Current tests cover:

- Creating a `BookChapter`
- Required title and slug generation behavior
- Adding chapters through the Application layer
- Reordering chapters
- Listing, adding, updating, deleting, and reordering chapters through the API integration test

## Known Limitations

- There is no standalone `/admin/book-chapters` list page.
- Chapter delete is a soft delete through `IsActive = false`.
- Reorder requires the submitted id list to exactly match the book's current chapter set.
- Chapter content is stored as markdown text only; there is no rich text editor or export pipeline yet.
