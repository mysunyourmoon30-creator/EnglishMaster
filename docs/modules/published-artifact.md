# Published Artifact

## Purpose

`PublishedArtifact` records output generated by a publish job. It links generated files back to their job, source content, and format.

## Fields

| Field | Description |
| --- | --- |
| `Id` | Unique artifact identifier. |
| `PublishJobId` | Parent publish job identifier. Required. |
| `SourceType` | Source content type: `Lesson`, `Course`, `Book`, or `Quiz`. |
| `SourceId` | Identifier of the source content record. |
| `Format` | Output format: `Html`, `Markdown`, `Pdf`, or `Docx`. |
| `FileName` | Generated file name. Required. |
| `FilePath` | Relative file path. Required for local files. |
| `PublicUrl` | Public URL for static file access when available. |
| `FileSize` | File size in bytes. Cannot be negative. |
| `ContentType` | Output content type. |
| `CreatedAt` | Creation timestamp. |
| `UpdatedAt` | Last update timestamp. |

## Domain Rules

- Artifact belongs to one `PublishJob`.
- `PublishJobId`, `SourceId`, `FileName`, `Format`, and `FilePath` are required.
- `FileSize` must not be negative.
- Internal absolute server paths are not exposed by DTOs.
