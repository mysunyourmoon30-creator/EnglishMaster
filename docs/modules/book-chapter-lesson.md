# Book Chapter Lesson

## Purpose

`BookChapterLesson` is the ordered join entity between `BookChapter` and `Lesson`. It lets a chapter contain many lessons while allowing the same lesson to appear in many chapters. It is not an independent business module and has no standalone Blazor route.

## BookChapterLesson Fields

| Field | Description |
| --- | --- |
| `Id` | Unique relation identifier. Generated by the domain and configured with `ValueGeneratedNever` in EF Core. |
| `BookChapterId` | Required parent chapter identifier. |
| `LessonId` | Required lesson identifier. |
| `LessonTitle` | Lesson title included in response DTOs. |
| `LessonSlug` | Lesson slug included in response DTOs. |
| `LessonSummary` | Lesson summary included in response DTOs. |
| `LessonCefrLevel` | Optional lesson CEFR level included in response DTOs. |
| `SortOrder` | Display order inside the chapter. Must be zero or greater. |
| `CreatedAt` | Relation creation timestamp. |
| `UpdatedAt` | Relation last update timestamp. |

## BookChapter Relationship

`BookChapterLesson` belongs to one `BookChapter`.

- `BookChapter` owns the `Lessons` collection.
- EF Core maps `BookChapter` to many `BookChapterLesson` rows with cascade delete from chapter to relation rows.
- The domain method `BookChapter.AddLesson` creates relation rows and prevents duplicates.
- `BookChapter.RemoveLesson` removes an existing relation and updates the chapter timestamp.

## Lesson Relationship

`BookChapterLesson` references one `Lesson`.

- The Application layer validates that the lesson exists and is active before it can be added.
- The database relationship uses restrict delete behavior for `LessonId`, so referenced lessons are protected.
- Response DTOs enrich the relation with lesson title, slug, summary, and CEFR level.

## Domain Rules

- `Id`, `BookChapterId`, and `LessonId` cannot be empty GUIDs.
- `SortOrder` must be zero or greater.
- `Reorder` changes `SortOrder` and updates `UpdatedAt`.
- Duplicate `LessonId` values inside one chapter are prevented by both domain behavior and a unique index on `(BookChapterId, LessonId)`.
- Domain code does not reference EF Core, ASP.NET Core, Blazor, or infrastructure.

## API Endpoints

| Method | Route | Purpose |
| --- | --- | --- |
| `GET` | `/api/v1/book-chapters/{chapterId}/lessons` | List lessons in a chapter ordered by `SortOrder`. |
| `POST` | `/api/v1/book-chapters/{chapterId}/lessons/{lessonId}` | Add a lesson to a chapter. Optional query parameter: `sortOrder`. |
| `DELETE` | `/api/v1/book-chapters/{chapterId}/lessons/{lessonId}` | Remove a lesson from a chapter. |
| `POST` | `/api/v1/book-chapters/{chapterId}/lessons/reorder` | Reorder all lessons in a chapter. |

Full status codes and examples are documented in [Book Chapter API](../api/book-chapter-api.md).

## Blazor UI

Chapter lessons are managed inline on `/admin/books/{id:guid}`.

- Admins select a chapter.
- Active lessons are loaded into a selector.
- Already-related lessons are hidden from the selector for the selected chapter.
- Admins can add lessons, remove lessons, and reorder lessons using Up and Down actions.
- Loading, empty, error, and validation states are shown through the Book detail page.

## Test Coverage

Current tests cover:

- Creating chapter lesson relations through `BookChapter.AddLesson`
- Duplicate lesson prevention inside one chapter
- Reordering `BookChapterLesson` rows
- Listing, adding, removing, and reordering chapter lessons through the API integration test

## Known Limitations

- There is no standalone `/admin/book-chapter-lessons` page.
- There is no standalone `GET /api/v1/book-chapter-lessons/{id}` route.
- Reorder requires the submitted id list to exactly match the chapter's current lesson relation set.
- Relation rows are removed from the chapter instead of soft-deactivated.
