# v0.2.0 Post-Release Monitoring

## Purpose

Track EnglishMaster v0.2.0 health after production deployment. This document is prepared before go-live and should be filled in by the assigned monitoring owner.

## First 24 Hours

| Check | Frequency | Status | Notes |
| --- | --- | --- | --- |
| App health endpoints | Every 30 minutes | Not Started | Verify live and ready endpoints. |
| Login success/failure | Every 2 hours | Not Started | Watch abnormal failure spikes. |
| Admin dashboard | Every 4 hours | Not Started | Confirm key admin page loads. |
| Public search | Every 4 hours | Not Started | Confirm public search returns safe results. |
| Student dashboard | Every 4 hours | Not Started | Confirm authenticated learner view works. |
| Quiz attempts | Every 4 hours | Not Started | Watch failures and answer exposure reports. |
| Practice sessions | Every 4 hours | Not Started | Confirm sessions complete. |
| Study plans | Every 4 hours | Not Started | Confirm plan generation/view works. |
| Import jobs | Every 4 hours | Not Started | Watch validation failures and unsafe filenames. |
| Publish jobs | Every 4 hours | Not Started | Confirm publishing jobs do not fail unexpectedly. |
| Email messages | Every 4 hours | Not Started | Confirm queued/sent/failed counts. |
| Notifications | Every 4 hours | Not Started | Confirm notification creation and reads. |
| Error logs | Hourly | Not Started | Review without exposing secrets. |
| Database health | Hourly | Not Started | Confirm connectivity and query health. |
| Disk space | Every 4 hours | Not Started | Include media and publishing storage. |
| Backup status | Daily | Not Started | Confirm backup job and restore readiness. |

## First 7 Days

| Check | Frequency | Status | Notes |
| --- | --- | --- | --- |
| Student activity | Daily | Not Started | Watch activity capture and dashboard trends. |
| Failed jobs | Daily | Not Started | Include imports, publishing, notifications, and email messages. |
| Performance issues | Daily | Not Started | Watch slow endpoints and database pressure. |
| Security issues | Daily | Not Started | Review auth failures and permission denials. |
| Data privacy reports | Daily | Not Started | Investigate any cross-user or exposed data report immediately. |
| Content review issues | Daily | Not Started | Watch review queue and restore request failures. |
| Import/export issues | Daily | Not Started | Review validation errors and operator feedback. |
| Public page errors | Daily | Not Started | Watch public route errors and search failures. |

## Escalation

Escalate immediately for data loss, private data exposure, public draft exposure, quiz answer exposure, authentication bypass, persistent outage, failed backups, or unrecoverable migration issues.
